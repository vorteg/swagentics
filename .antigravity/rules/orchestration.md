# RULE: Orchestration & Determinism

<orchestration_protocol>
  **MANDATORY BEHAVIOR:** Antigravity MUST rely on deterministic Python/Bash scripts for repository mapping and state generation. DO NOT attempt to guess the repository structure or manually index files unless explicitly instructed by a script's output.

  1. **Read State First:** Always begin workflows by reading the `.github/memory/active_state.tson` generated by the hook scripts (e.g., `blueprint.py`).
  2. **Trust the Hook:** If a task requires knowing modified files or active skills, run the corresponding script in `.github/hooks/scripts/` and consume its output.
  3. **No Hallucinations:** If a path or variable is missing from the `.tson` state, stop and ask the user to run the appropriate sync script.
</orchestration_protocol>

<model_routing>
  Optimize request usage by selecting the appropriate model for the task:
  - **Lightweight Tasks:** Use Flash/Light models (e.g., Gemini Flash) for static analysis, parsing logs, or simple test generation.
  - **Heavyweight Tasks:** Use Pro/Sonnet models (e.g., Gemini Pro, Claude Sonnet) for architecture, complex bug fixing, and Core logic implementation.
  - **AVOID:** Do not use legacy or highly hallucinatory models (e.g., GPT-3.5 or unverified generic models) for Swagentics execution.
</model_routing>
